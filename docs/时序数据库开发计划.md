## 1. 功能概述

### 1.1 目标

为晶晶助手集成时序数据库，实现：
- **指标存储**：记录 API 调用、响应时间、错误率等时间序列数据
- **可视化查询**：支持按时间范围查询和聚合统计
- **告警基础**：为后续告警功能提供数据支撑

### 1.2 使用场景

| 场景 | 数据类型 | 查询需求 |
|------|----------|----------|
| API 监控 | 响应时间、状态码 | 最近 1 小时平均响应时间 |
| 使用统计 | 调用次数、用户数 | 每日/每周调用趋势 |
| 工具分析 | 工具调用频次 | 最常用的工具 TOP 5 |
| 错误追踪 | 错误类型、频次 | 错误率趋势 |

## 2. 技术选型

### 2.1 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **InfluxDB** | 生态完善、文档丰富 | 需要单独部署 | ⭐⭐⭐⭐⭐ |
| **TimescaleDB** | 基于 PostgreSQL | 需要 PG 基础 | ⭐⭐⭐⭐ |
| **SQLite + 自定义** | 无额外依赖 | 性能有限 | ⭐⭐⭐ |
| **Prometheus** | 监控专用 | 配置复杂 | ⭐⭐⭐ |

### 2.2 推荐方案：InfluxDB 2.x

**选择理由**：
1. 专为时序数据设计，写入和查询性能优秀
2. 内置 Web UI，方便数据可视化
3. 支持 Docker 部署，与现有架构兼容
4. Python 客户端成熟稳定

**版本**：InfluxDB 2.7+（OSS 开源版）

#### 5.2.2 Python 客户端

```python
# src/metrics/client.py
from influxdb_client import InfluxDBClient, Point
from influxdb_client.client.write_api import SYNCHRONOUS

class MetricsClient:
    def __init__(self, url, token, org, bucket):
        self.client = InfluxDBClient(url=url, token=token, org=org)
        self.write_api = self.client.write_api(write_options=SYNCHRONOUS)
        self.query_api = self.client.query_api()
        self.bucket = bucket
        self.org = org
    
    def write_api_request(self, endpoint, method, status_code, duration_ms, client):
        point = Point("api_requests") \
            .tag("endpoint", endpoint) \
            .tag("method", method) \
            .tag("status_code", str(status_code)) \
            .tag("client", client) \
            .field("duration_ms", duration_ms)
        self.write_api.write(bucket=self.bucket, record=point)
    
    def query_avg_response_time(self, hours=1):
        query = f'''
        from(bucket: "{self.bucket}")
        |> range(start: -{hours}h)
        |> filter(fn: (r) => r._measurement == "api_requests")
        |> filter(fn: (r) => r._field == "duration_ms")
        |> mean()
        '''
        return self.query_api.query(query, org=self.org)
```

## 7. 接口设计

### 7.1 GET /api/metrics/summary

获取指标摘要。

**响应示例**：
```json
{
  "period": "last_24h",
  "total_requests": 1234,
  "total_chats": 456,
  "avg_response_time_ms": 150.5,
  "error_rate": 0.02,
  "top_endpoints": [
    {"endpoint": "/api/chat", "count": 300},
    {"endpoint": "/api/knowledge", "count": 150}
  ],
  "top_tools": [
    {"tool": "search_knowledge_base", "count": 200},
    {"tool": "get_weather", "count": 100}
  ]
}
```

### 7.2 GET /api/metrics/trends

获取趋势数据。

**参数**：
- `metric`: 指标名称（requests, response_time, errors）
- `period`: 时间范围（1h, 24h, 7d, 30d）
- `interval`: 聚合间隔（1m, 5m, 1h, 1d）

**响应示例**：
```json
{
  "metric": "requests",
  "period": "24h",
  "interval": "1h",
  "data": [
    {"time": "2026-03-09T00:00:00Z", "value": 45},
    {"time": "2026-03-09T01:00:00Z", "value": 32},
    ...
  ]
}
```

## 10. 风险与应对

| 风险 | 影响 | 应对措施 |
