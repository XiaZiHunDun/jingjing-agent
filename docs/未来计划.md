### 1.1 时序数据库集成

**为什么需要：** Agent运行会产生大量时间序列数据（对话历史、性能指标、用户行为），普通数据库查询慢。

**选型：**

| 方案 | 特点 | 你的场景 |
|------|------|---------|
| **InfluxDB** | 最流行，生态好 | 推荐，文档多 |
| **TimescaleDB** | PostgreSQL插件 | 已有PG基础选 |
| **TDengine** | 国产，高性能 | 物联网/大数据量 |

**最小可用集成（InfluxDB）：**
```python
from influxdb_client import InfluxDBClient, Point
from influxdb_client.client.write_api import SYNCHRONOUS

class AgentMetrics:
    def __init__(self):
        self.client = InfluxDBClient(url="http://localhost:8086", token="my-token")
        self.write_api = self.client.write_api(write_options=SYNCHRONOUS)
    
    def log_query(self, user_id: str, query: str, latency: float, success: bool):
        """记录每次查询的指标"""
        point = Point("agent_queries") \
            .tag("user_id", user_id) \
            .tag("success", str(success)) \
            .field("query_length", len(query)) \
            .field("latency_ms", latency) \
            .field("token_count", self.estimate_tokens(query))
        self.write_api.write(bucket="agent_bucket", record=point)
    
    def get_stats(self, hours=24):
        """查询最近24小时统计"""
        query = f'''
        from(bucket: "agent_bucket")
        |> range(start: -{hours}h)
        |> filter(fn: (r) => r._measurement == "agent_queries")
        |> aggregateWindow(every: 1h, fn: mean)
        '''
        return self.client.query_api().query(query)
```

**验收标准：**
- [ ] 能记录每次Agent调用的延迟、Token数、成功率
- [ ] 能查询"最近1小时平均响应时间"等基础指标
- [ ] 数据保留策略：自动清理7天前数据

## 阶段2：本地模型接入（阶段1完成后1-2周）

### 2.1 模型选型（基于你的Agent类型）

**你的Agent是哪种？**

| Agent类型 | 推荐模型 | 显存需求 | 理由 |
|----------|---------|---------|------|
| 通用问答/知识库 | Qwen2.5-7B-Instruct | 10-12G | 中文最强7B |
| 推理/分析类 | DeepSeek-R1-Distill-7B | 12-14G | 推理能力突出 |
| 代码/技术类 | Qwen2.5-Coder-7B | 10-12G | 代码专项优化 |
| 多模态（图文） | Qwen2-VL-7B | 12-14G | 能看图说话 |

### 2.2 接入方案（渐进式）

**方案A：双模型架构（推荐，风险低）**
```
用户请求 → 路由层 → 简单问题 → 本地7B模型
                → 复杂问题 → API大模型（GPT-4/Claude）
```

**代码实现：**
```python
class HybridAgent:
    def __init__(self):
        self.local_llm = Ollama(model="qwen2.5:7b")
        self.api_llm = ChatOpenAI(model="gpt-4")
        self.router = self.build_router()
    
    def route(self, query: str) -> str:
        """判断用哪个模型"""
        # 简单启发式：长度+关键词
        if len(query) < 50 and "简单" in query:
            return "local"
        return "api"
    
    async def chat(self, query: str):
        route = self.route(query)
        if route == "local":
            return await self.local_llm.ainvoke(query)
        else:
            return await self.api_llm.ainvoke(query)
```

**方案B：完全本地（成本低，效果可能降）**
- 直接替换API调用为Ollama
- 需要调整Prompt（7B模型指令跟随能力弱）

### 2.3 效果评估（关键！）

**建立评估集（现在就要做）：**
```json
[
  {
    "query": "我们公司请假流程是什么？",
    "expected": "根据员工手册第3章，请假需提前1天在OA系统提交...",
    "category": "factual"  // 事实性问题
  },
  {
    "query": "分析Q3销售数据下降的原因",
    "expected": "需要结合时序数据库中的销售指标...",
    "category": "analytical"  // 分析性问题
  }
]
```

**自动化评估脚本：**
```python
from difflib import SequenceMatcher

def evaluate(agent, test_cases):
    results = []
    for case in test_cases:
        response = agent.chat(case["query"])
        # 简单相似度（实际可用GPT-4打分）
        score = SequenceMatcher(None, response, case["expected"]).ratio()
        results.append({
            "query": case["query"],
            "score": score,
            "response": response
        })
    return results

# 对比API vs 本地模型
api_scores = evaluate(api_agent, test_cases)
local_scores = evaluate(local_agent, test_cases)

print(f"API平均得分: {sum(s['score'] for s in api_scores)/len(api_scores)}")
print(f"本地平均得分: {sum(s['score'] for s in local_scores)/len(local_scores)}")
```

**阶段2验收标准：**
- [ ] 本地模型能跑通Agent全流程（哪怕效果差）
- [ ] 有量化评估：本地模型 vs API模型的差距
- [ ] 确定是否值得微调（差距<30%才值得调）

### 3.3 评估与迭代

**A/B测试框架：**
```python
def ab_test(query, model_a, model_b, evaluator="gpt-4"):
    """用GPT-4作为裁判，判断哪个回答更好"""
    resp_a = model_a.chat(query)
    resp_b = model_b.chat(query)
    
    judge_prompt = f"""
    请判断哪个回答更好：
    问题：{query}
    回答A：{resp_a}
    回答B：{resp_b}
    从准确性、完整性、专业性评分。只输出A或B。
    """
    result = evaluator.invoke(judge_prompt)
    return result, resp_a, resp_b
```

## 关键决策点

### 决策1：阶段1必须做多重？
**建议：** 至少完成Docker部署和基础监控。时序库可以先放，用SQLite/PostgreSQL顶一顶。

### 决策2：什么情况下放弃本地模型？
- 如果7B模型在你的场景下效果极差（<API的50%）
- 如果Agent需要很强的推理/代码能力（考虑14B或放弃本地）

### 决策3：微调值得吗？
**算笔账：**
- 微调成本：云GPU 500元 + 数据标注 20小时
- 收益：外包报价可+3-5万，或节省API费用 500元/月
- **ROI：** 接1个项目就回本，值得做
